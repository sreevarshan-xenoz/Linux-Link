🚀 Linux Link v2.0: The Unified QUIC Evolution

Blueprint for the next generation of Linux Link. Focus: Extreme Performance, Zero-Config Convenience, and Deep Desktop Integration.

The Motive

The current architecture relies on a "Split-Brain" model: TCP (KDE Connect) for control and QUIC for streaming. This causes latency inconsistencies, complex firewall requirements, and state synchronization bugs. By unifying everything under a single multiplexed QUIC tunnel, we achieve lower latency, robust connection roaming (switching from WiFi to 5G seamlessly), and the architectural foundation needed for advanced features like 120FPS streaming and virtual displays.

✨ Feature Upgrades

⚡ Performance

  • 120FPS Support: Dynamic frame pacing and high-refresh-rate sync for butter-smooth UI.
  • AV1 Encoding: 30% bandwidth reduction for supported hardware (Intel Arc, AMD 7000, RTX 40-series).
  • Connection Roaming: Instantly resume sessions when switching from WiFi to Cellular (thanks to QUIC Connection IDs).

🛠 Deep Desktop Integration

  • Virtual Monitor: Server spawns a ghost display (Wayland `wlr-virtual-pointer` / X11 Dummy) so the phone acts as a true second screen.
  • Magic Drag & Drop: Share a file on Android to instantly "paste" it into the active Linux window.
  • Macro-Pad Mode: Use the Android screen as a customizable touch-bar for Linux shortcuts.

🔒 Zero-Config Convenience

  • QR Pairing: Scan a terminal QR code to securely exchange TLS certificates instantly.
  • mDNS Auto-Discovery: No more IP typing; servers broadcast natively on the LAN.
  • Wake-on-LAN: Wake up sleeping machines directly from the app interface.

🚀 Implementation Phases

  1. Phase 1 (The Foundation): Rip out KDE Connect logic. Implement the unified QUIC multiplexer on the Rust server.
  2. Phase 2 (The Bridge): Update the Android Rust FFI and Dart services to handle the new stream-based protocol.
  3. Phase 3 (Productivity): Implement Virtual Monitor and Magic Drag & Drop.
  4. Phase 4 (Polish): Add AV1 support, 120FPS pacing, and QR-based Zero-Config pairing.
